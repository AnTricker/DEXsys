# 前端增加新功能

> 在LINE記事本，用戶可以隨時查閱以前的紀錄 => 這邊也要加入

### 本版本實作部分
    - UI：
        - homepage：前端現有「點名」、「銷售」，再多一個「查詢」
        - 進入「查詢」頁面需要密碼，其設定在 "Settings"的"SearchPwd"
        - 「查詢」輸入：上方「選擇年月」（下拉式）、下方「選擇教練」（下拉式，資料從“Teachers”表抓取），最下方「查詢」按鈕
        - 「查詢」輸出：條列式顯示其登記過的「點名」＆「銷售」資訊
            - 輸出格式：按照日期順序上下排列即可，若超過手機頁面範圍，設計「向下滑動」即可
                - 點名：日期｜課程名稱｜上課人數
                - 銷售：日期｜銷售名目｜數量｜總金額
        - 整體css按照目前風格
    - 「查詢」的UX:
        - 用戶於homepage點擊「查詢」，進入輸入頁
        - 用戶輸入完月份＆教練後，點擊「查詢」
        - 顯示該月份的「點名」和「銷售」紀錄（去"Attendances"&"Sales"表抓資料，格式如上所示）       

### 補充細節
    - 當同一天既有「點名」又有「銷售」時，點名在前、銷售在後。
    - 點擊查詢後應有 Loading... 狀態。
    - 若該教練該月無紀錄，應顯示「本月尚無紀錄」而非一片空白。
